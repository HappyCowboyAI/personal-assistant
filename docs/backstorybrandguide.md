## Brand Philosophy & Voice

Backstory is the system of record for modern revenue teams. We gather activity across tools and channels, interpret it in context, and produce a reliable account timeline teams can trust.

### Who We Are

| We Are | We're Not |
|--------|-----------|
| Clear — we make complex data simple | Just Dashboards — we give context, not charts |
| Interpretive — we give data meaning | Abstract or Academic — we speak plainly |
| Engaging — we make revenue compelling | Noisy — we surface signal, not volume |
| Grounded — built on reliable data | Speculative — we report what happened |
| Trustworthy — accurate when it matters | Sales Theater — we back claims with data |
| Action-Oriented — we surface what matters | Hard to Use — clarity is a product requirement |

### Voice Principles

- **Clear** — We say what we mean. No jargon, no filler. If a simpler word works, use it.
- **Confident** — We know our product and trust it. We don't hedge unnecessarily.
- **Engaging** — We make the complex interesting. Data has a story — we help tell it.
- **Direct** — We get to the point. Brevity is a feature.
- **Approachable** — Smart but not academic. Serious but not stiff.

### Social Voice
Write the way we talk: direct, precise, and with a point of view. Lead with the idea, not the product. Use data to back claims. Avoid jargon and filler.  
Hashtags: `#SalesAI` `#RevOps` `#PipelineIntelligence`

## Brandmarks & Logo Usage

The Backstory brandmark consists of two elements: the **Symbol** (stylized books/files icon) and the **Wordmark**. Together they form the **Logo**.

### Available Lockups

| Lockup | Use Case |
|--------|----------|
| Symbol + Wordmark (primary) | All primary brand moments: website, marketing, email, presentations |
| Horizontal lockup | Headers, banners, and horizontal layouts |
| Symbol alone | Social avatars, app icons, space-constrained applications |
| Wordmark alone | Special editorial contexts only |

### Logo Versions

The logo appears in **two versions only**:
- **Black** — on light backgrounds
- **White** — on dark backgrounds

No color versions, gradients, outlines, shadows, or effects are permitted.

### Exclusion Zone

Maintain clear space around the logo equal to **half the logo height** on all sides.

### Minimum Sizes

| Element | Digital | Print |
|---------|---------|-------|
| Symbol only | 32px | 10mm |
| Full logo (Symbol + Wordmark) | 80px | 30mm |

### Logo Rules

**Always:**
- Use approved black or white versions only
- Maintain the required exclusion zone
- Respect minimum size requirements
- Use on on-brand background colors only

**Never:**
- Recolor, tint, or apply gradients to the logo
- Stretch, skew, distort, or rotate the logo
- Apply drop shadows, glows, or emboss effects
- Use on colors outside the brand color palette
- Recreate or modify any letterforms
- Use below minimum sizes
- Place on busy photographic backgrounds without sufficient contrast

## Color

Our brand is built first on **black and white**. Clarity, contrast, and restraint define the system. Color is used sparingly and with intent.

**Horizon functions as an accent, not a foundation.** It exists to support hierarchy, emphasis, and moments of focus — never to dominate the composition.

### Color Hierarchy / Weighting

| Weight | Colors | Usage |
|--------|--------|-------|
| **Neutral (dominant)** | Black, White | Primary visual language — always lead with these |

## Data Visualization

Data visualization is a core brand expression. Charts and dashboards reflect our commitment to clarity, precision, and interpretability.

### Data Visualization Principles

**Do:**
- Use typography and whitespace as primary organizational tools
- Apply brand colors only to highlight what matters most
- Ensure sufficient contrast for accessibility
- Use **Chivo Mono** for all numerical labels and data callouts
- Keep chart backgrounds neutral (White or Graphite) — never colored
- Use Data/1–6 semantic tokens for chart coloring

**Don't:**
- Use the full secondary palette as a data palette
- Apply branding to every data point
- Use gradients within chart elements
- Sacrifice readability for brand expression

### Data Color Order

For multi-series charts, use in order: Horizon → Mint → Plum → Cobalt → La Te light → La Te dark.

## Imagery

### Photography Direction

- Clean, textural imagery to create depth without overpowering layout
- Minimal and intentional; allow structure, contrast, and shadow to do the work
- Favor natural textures and quiet, focused compositions
- Avoid generic stock photography or heavily posed shots

### Painting Elements

Backstory may incorporate painterly public domain artwork as a background element. These works introduce a classical reference that supports the brand's literary and narrative character. Always transform — never reproduce untreated.

### Image Treatment

Use reeded glass or subtle refraction to mediate public-domain artwork. The image should feel **viewed through a surface**, not filtered.

### Imagery Rules

**Do:**
- Favor natural light and organic textures
- Allow contrast and shadow to create depth
- Transform public domain art — never use untreated
- Ensure the image supports the content, never competes with it

**Don't:**
- Use stock imagery that feels generic or posed
- Apply color filters or heavy effects to photography
- Use imagery that introduces off-brand color
- Reproduce public domain artwork without transformation
- Place text over imagery without sufficient contrast

## Presentation System (PPTX)

Backstory presentations are built with **pptxgenjs** and follow the brand guidelines with font fallbacks where licensed typefaces are unavailable in PPTX viewers.

### Font Mapping for PPTX

| Brand Typeface | Role | PPTX Fallback |
|---------------|------|---------------|
| LL Kleisch Light | Headlines, display | `"Cardo"` (serif; embed LL Kleisch if licensed) |
| KMR Waldenburg Regular | Body, UI, labels | `"Roboto"` (sans-serif; embed KMR Waldenburg if licensed) |
| Chivo Mono | Data labels, figures | `"Chivo Mono"` (available on Google Fonts — embed directly) |

> To use the official brand typefaces in PPTX, embed the font files. Without embedding, fallback fonts render instead on viewer systems without the fonts installed. Chivo Mono is the most portable — it is available via Google Fonts and renders consistently.

### Critical PPTX Rules

1. **Single text blocks** for all bulleted lists — never separate text boxes per bullet
2. **`calcFontSize()`** for all card/column labels — pptxgenjs does not auto-shrink text
3. **Dark icon on light backgrounds; white icon on dark backgrounds** — never reverse
4. **Chivo Mono** for stat values and numerical callouts on dashboard slides
5. **Gradient stripe** should always use white books icon (confirmed true PNG)
6. **Building icon** files sourced from PPTX extraction may be JPEG with `.png` extension — verify first 4 bytes are `89504e47` before use
